# EV-010 Downstream Contract Event and Snapshot Report

Date: 2026-05-27
Phase: PH-08 / commit-08-a
Gate: GATE-T-07

## Scope

- Verify fake downstream consumers can build local definition indexes from published events or exported snapshots.
- Verify consumers do not write definition truth back into `method-library`.
- Verify published event envelopes remain consumable for downstream sync.

## Covered Tests

| TC ID | Test | Coverage |
|---|---|---|
| TC-SYNC-001 | `syncs_identity_catalog_from_qualification_and_role_events` | Identity catalog builds from published `Qualification` and `RoleDefinition` events and keeps fingerprints. |
| TC-SYNC-002 | `syncs_process_definition_indexes_without_creating_instances` | Process definition indexes consume `TaskDefinition` and `ProcessTemplateDef` events without creating process instances. |
| TC-SYNC-003 | `uses_qualification_snapshots_as_capability_binding_anchors` | Capability-hub uses `Qualification` snapshot as an anchor and stores only binding references. |
| TC-SYNC-004 | `syncs_artifact_definition_mappings_without_creating_instances` | Artifact consumer updates work-product mappings from snapshots without creating artifact instances. |
| TC-SYNC-005 | `syncs_governance_policy_sources_without_enforce_results` | Governance consumer tracks `AIPolicyDef` published sources without producing enforce results. |
| TC-SYNC-006 | `resolves_console_view_profiles_without_local_authority_rules` | Console consumer observes published `ViewProfile` and resolves authority from `ResolveViewProfile`. |
| TC-EVT-001 | `syncs_identity_catalog_from_qualification_and_role_events` | Published event envelope carries `event_id`, `schema_version`, `content_ref`, `snapshot_ref`, and trace context. |
| TC-WRK-001 | `syncs_identity_catalog_from_qualification_and_role_events` | Outbox relay publishes pending events to the definition topic before downstream consumption. |
| TC-STATE-003 | `syncs_identity_catalog_from_qualification_and_role_events` | Relayed outbox events transition to `published`. |

## Execution

```bash
cargo test -p method_library_application --test downstream_contracts
```

## Evidence Notes

- Event-path consumers read the original published event IDs from the fake bus and persist local fingerprints.
- Snapshot-path consumers export immutable definition snapshots with fingerprint verification enabled.
- Every contract test captures write-model counts before and after consumer sync to prove no reverse writes into `method-library`.
